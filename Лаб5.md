1. Проверим ещё раз соединение между оконечными узлами. Все пакеты потеряны. Кольцо на хабах/коммутаторах без протокола STP создает широковещательный шторм и выводит сеть из строя. Это сетевой сбой, при котором широковещательные пакеты бесконтрольно и лавинообразно размножаются в сети, загружая канал связи до 100% и полностью парализуют её работу. Для решения этой проблемы используется протокол STP.

![Лаб5](<Основы%20сетевых%20технологий/Attachments/Лаб5%207.png>)

1. Заменим концентраторы на коммутаторы, чтобы решить проблему выше.

1. Протестируем связь ещё раз. Все пакеты были получены. Коммутаторы поддерживают **STP (Spanning Tree Protocol)**, который блокирует избыточные связи и предотвращает петли.

1. Узнаем, какое соединение с корневым коммутатором отключено. Нормальное соединение: Root Path Cost = 4-19 (маленькое значение)

Отключенное/заблокированное соединение: Root Path Cost = 100+ (большое значение).

Оба соединения LSW4-LSW2 и LSW4-LSW3. Root cost у них равен 200000.

Расчет кратчайших путей от LSW4 к корню LSW5:

Вариант 1: Прямой путь LSW4 → LSW5

- Cost = 19 (если бы соединение было активно)

Вариант 2: Через LSW2: LSW4 → LSW2 → LSW5

- Cost = 200000 + 19 = 200019

Вариант 3: Через LSW3: LSW4 → LSW3 → LSW5

- Cost = 200000 + 19 = 200019
