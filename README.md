# Отчеты по лабораторным работам: Основы сетевых технологий

## Содержание лабораторных работ
* **Дополнительно (Lab4.pkt):** В файле симуляции Cisco Packet Tracer реализована настройка виртуальных локальных сетей (VLAN) для логического разделения трафика. Также для обеспечения связности между изолированными подсетями была настроена маршрутизация: статическая для постоянных маршрутов и динамическая для автоматического обмена маршрутной информацией между устройствами.
* **Лабораторная работа №1: Основные понятия сетевых технологий**
  Знакомство с симуляторами сетей (Cisco Packet Tracer, Huawei eNSP) и системами мониторинга (Zabbix, Nagios). Построение простейшей локальной сети на базе коммутатора, базовое использование Wireshark и разбор инкапсуляции в модели OSI.
* **Лабораторная работа №2: Кабельные системы локальных вычислительных сетей**
  Проектирование физической топологии сети с использованием Wi-Fi и оптоволокна. Изучение инструментов для работы с оптическим кабелем (скалыватели, рефлектометры) и сравнительный анализ возможностей Packet Tracer и eNSP.
* **Лабораторная работа №3: Настройка среды моделирования и простейшая ЛВС**
  Построение сети "точка-точка" с настройкой статических IP-адресов. Захват и анализ ICMP-трафика в Wireshark, а также программное определение MAC-адресов локальных сетевых интерфейсов.
* **Лабораторная работа №4: Начальная конфигурация коммутатора и маршрутизатора**
  Базовая настройка оборудования Huawei (установка имени, времени, сохранение конфигурации) и запуск DHCP-сервера на маршрутизаторе AR1220. 
* **Лабораторная работа №5: Работа с коммутаторами. Протокол STP**
  Изучение явления «широковещательного шторма». Настройка Spanning Tree Protocol (STP) для предотвращения петель маршрутизации, определение корневого коммутатора (Root Bridge) и анализ стоимости путей.
* **Лабораторная работа №6: Диагностика Wi-Fi и маршрутизация**
  Анализ загруженности радиоэфира (2.4 и 5 ГГц) с помощью inSSIDer. Расчет масок подсетей, работа с локальной таблицей маршрутизации Windows (добавление статических маршрутов) и использование утилит `ping` и `tracert`.
* **Лабораторная работа №7: Управляющие протоколы сетевого уровня**
  Анализ протоколов с помощью Wireshark. Изучение процесса выдачи адресов DHCP (DORA), структуры ARP-запросов/ответов и генерации ICMP-ошибок (превышение TTL).
* **Лабораторная работа №8: Протоколы транспортного уровня TCP/IP**
  Сбор сетевой статистики и просмотр активных подключений с помощью `netstat`. Детальный разбор механизма установки (тройное рукопожатие) и разрыва TCP-соединения на уровне флагов и порядковых номеров.
* **Лабораторная работа №9: Инкапсуляция протоколов и сокеты**
  Создание клиент-серверного приложения на Python с использованием сокетов и анализ его трафика через loopback-интерфейс. Настройка брандмауэра Windows через консольную утилиту `netsh` (экспорт/импорт конфигураций, открытие портов).
* **Лабораторная работа №10: Прикладной уровень (DNS и HTTP)**
  Изучение структуры DNS-запросов утилитой `nslookup` (включая работу с корневыми серверами) и локального файла `hosts`. Анализ HTTP-трафика: перенаправления (коды 301/302) и ошибки доступа (404).
* **Лабораторная работа №11: Диагностика сетей на сетевом уровне**
  Отработка навыков поиска маршрута пакетов и проверки связности сети. Работа с CLI маршрутизатора для перенастройки IP-адресов интерфейсов.
* **Лабораторная работа №12: Трассировка маршрутов и удаленный доступ**
  Диагностика сложной сетевой топологии с помощью `tracert`. Удаленное подключение к маршрутизаторам по Telnet для анализа их таблиц маршрутизации (`show ip route`) и восстановления схемы адресации.
* **Лабораторная работа №13: Выявление и устранение неполадок маршрутизации**
  Поиск и устранение маршрутных петель и ошибочных статических маршрутов на нескольких устройствах. Внесение корректив в таблицы маршрутизации для восстановления связи с целевым сервером.
* **Лабораторная работа №14: Широковещательный трафик в сетях VLAN**
  Анализ поведения коммутаторов при обработке широковещательных пакетов в сетях с VLAN и без него. Изучение понятий коллизионных доменов и изолированных широковещательных доменов.
