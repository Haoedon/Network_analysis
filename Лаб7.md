**DORA** (Discover, Offer, Request, Acknowledgement) — это последовательность из четырёх шагов, которые используются клиентами и серверами DHCP для назначения IP-адресов.

- **Discover**. Клиент ещё не имеет IP-адреса, поэтому в качестве источника используется 0.0.0.0. Назначение пакета — 255.255.255.255, чтобы широковещательно достичь всех устройств локальной сети.
- **Offer**. Источник пакета — IP сервера DHCP, а назначение — IP, который сервер предлагает клиенту. На этом уровне сервер предлагает клиенту адрес, который тот сможет использовать, как только подтвердит выбор.
- **Request**. Источник пакета — 0.0.0.0, так как клиент ещё не получил окончательно назначенный IP. Назначение — 255.255.255.255 для доставки пакета всем устройствам локальной сети.
- **Acknowledgement**. После того как клиент отправил Request, сервер подтверждает окончательный выбор IP-адреса и сетевых параметров с помощью DHCP ACK — четвёртого и последнего шага процесса DORA.

**Расшифровка полей ARP-пакета (Reply):**

1. **Hardware type: Ethernet (1)**
    - **Значение:** 1
    - **Расшифровка:** Тип используемого оборудования на канальном уровне. Значение 1 стандартно обозначает технологию **Ethernet** (также может использоваться для IEEE 802.11 Wi-Fi).
2. **Protocol type: IPv4 (0x0800)**
    - **Значение:** 0x0800
    - **Расшифровка:** Тип сетевого протокола, для которого выполняется разрешение адресов. Значение 0x0800 соответствует протоколу **IPv4**.
3. **Hardware size: 6**
    - **Значение:** 6
    - **Расшифровка:** Размер аппаратного (физического) адреса в байтах. Для Ethernet MAC-адрес имеет длину **6 байт** (48 бит).
4. **Protocol size: 4**
    - **Значение:** 4
    - **Расшифровка:** Размер логического (сетевого) адреса в байтах. Для IPv4-адреса длина составляет **4 байта** (32 бита).
5. **Opcode: reply (2)**
    - **Значение:** 2
    - **Расшифровка:** Код операции, который определяет тип ARP-сообщения. Значение 2 означает, что это **ARP-Reply** (ответ). Устройство, которое запрашивали, отправляет этот пакет, содержащий свой MAC-адрес.
6. **Sender MAC address: ASRockIncorp_85:73:c1 (9c:6b:00:85:73:c1)**
    - **Значение:** 9c:6b:00:85:73:c1
    - **Расшифровка:** **MAC-адрес отправителя** этого ARP-сообщения. В данном случае это устройство, которое отвечает на запрос. Производитель сетевого интерфейса (определён по первым трём байтам OUI) — ASRock Incorporation. Это и есть тот самый адрес, который искался исходным запросом.
7. **Sender IP address: 192.168.0.76**
    - **Значение:** 192.168.0.76
    - **Расшифровка:** **IP-адрес отправителя** этого сообщения. Это адрес устройства, чей MAC-адрес был запрошен.
8. **Target MAC address: TPlink_83:ea:a8 (78:8c:b5:83:ea:a8)**
    - **Значение:** 78:8c:b5:83:ea:a8
    - **Расшифровка:** **MAC-адрес получателя** этого ARP-сообщения. Это устройство, которое изначально отправило ARP-request. Производитель сетевого интерфейса — TP-LINK TECHNOLOGIES CO., LTD. В поле запроса это поле было бы заполнено нулями (00:00:00:00:00:00), но в ответе отправитель уже знает MAC истца и указывает его здесь.
9. **Target IP address: 192.168.0.1**
    - **Значение:** 192.168.0.1
    - **Расшифровка:** **IP-адрес получателя** этого сообщения. Это адрес устройства, которое отправило запрос. Обычно это шлюз по умолчанию, который запрашивал MAC-адрес хоста 192.168.0.76.
